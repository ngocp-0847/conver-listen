# conversation-tts skill

Generate a two-speaker English dialogue about a user-supplied topic, emit a self-contained Python file + WAV, then play the WAV.

## When to invoke
The user wants to **hear** a generated dialogue. Triggers:
- "tôi muốn nghe 1 đoạn hội thoại về ..."
- "generate a conversation about ..."
- "phát cho tôi 1 cuộc thoại giữa X và Y"
- "đọc cho tôi 1 dialogue về ..."

If the user only wants a written dialogue with no audio, do NOT use this skill.

## Pre-flight check
The skill assumes `mvp/` exists at the project root with an initialized `uv` venv containing `supertonic`, `soundfile`, `numpy`. If `mvp/pyproject.toml` is missing, stop and tell the user to set up the MVP first.

## Steps (execute in order)

### 1. Make a slug
Translate the topic to a short English label, lowercase ASCII, hyphens, no diacritics.

Examples:
- "phỏng vấn senior developer" → `senior-developer-interview`
- "đặt phòng khách sạn" → `hotel-booking`
- "negotiating a job offer" → `negotiating-a-job-offer`

### 2. Generate the dialogue
Write a natural 10-14 turn English dialogue between two speakers. Pick speaker labels that fit the topic (`interviewer`/`candidate`, `customer`/`staff`, `host`/`guest`, `doctor`/`patient`, etc.). Each turn 1-3 sentences. Cover a meaningful arc: opening → substance → close.

### 3. Pick voices
Two different Supertonic voices, one per speaker. Available: `M1`-`M5`, `F1`-`F5`. Default to alternating gender for clarity (e.g. `M1` + `F1`).

### 4. Write the Python file
Create `mvp/outputs/{slug}/{slug}.py` using **exactly** this template — substitute only the five placeholders (`__TITLE__`, `__SPEAKER_A__`, `__SPEAKER_B__`, `__VOICE_A__`, `__VOICE_B__`, `__DIALOGUE__`):

```python
"""__TITLE__ — generated by conversation-tts skill."""
from __future__ import annotations
from pathlib import Path
import numpy as np
import soundfile as sf
from supertonic import TTS

OUTPUT = Path(__file__).with_suffix(".wav")
SAMPLE_RATE = 44100
PAUSE_MS = 350

SPEAKER_A = "__SPEAKER_A__"
SPEAKER_B = "__SPEAKER_B__"
VOICE_A = "__VOICE_A__"
VOICE_B = "__VOICE_B__"

DIALOGUE: list[tuple[str, str]] = __DIALOGUE__


def main() -> None:
    print("[init] Loading Supertonic...", flush=True)
    tts = TTS(auto_download=True)
    styles = {
        SPEAKER_A: tts.get_voice_style(voice_name=VOICE_A),
        SPEAKER_B: tts.get_voice_style(voice_name=VOICE_B),
    }
    silence = np.zeros((1, int(SAMPLE_RATE * PAUSE_MS / 1000)), dtype=np.float32)
    chunks: list[np.ndarray] = []
    for i, (speaker, line) in enumerate(DIALOGUE, 1):
        preview = line[:60] + ("..." if len(line) > 60 else "")
        print(f"[{i:02d}/{len(DIALOGUE)}] {speaker}: {preview}", flush=True)
        wav, _ = tts.synthesize(
            text=line, lang="en",
            voice_style=styles[speaker],
            total_steps=8, speed=1.05,
        )
        if wav.ndim == 1:
            wav = wav.reshape(1, -1)
        chunks.append(wav.astype(np.float32))
        chunks.append(silence)
    full = np.concatenate(chunks, axis=1)
    OUTPUT.parent.mkdir(parents=True, exist_ok=True)
    sf.write(str(OUTPUT), full.squeeze(), SAMPLE_RATE, subtype="PCM_16")
    print(f"[done] {full.shape[1] / SAMPLE_RATE:.1f}s -> {OUTPUT}")


if __name__ == "__main__":
    main()
```

Rules for the placeholders:
- `__TITLE__`: human-readable title in English (e.g. "Senior developer interview").
- `__SPEAKER_A__` / `__SPEAKER_B__`: short lowercase labels — must match the speaker keys used in `__DIALOGUE__`.
- `__VOICE_A__` / `__VOICE_B__`: two distinct strings from {M1,M2,M3,M4,M5,F1,F2,F3,F4,F5}.
- `__DIALOGUE__`: a literal Python list of `(speaker, text)` tuples — NOT a string. Example:
  ```python
  [
      ("interviewer", "Hi, thanks for coming in. Could you walk me through your background?"),
      ("candidate",   "Sure. I have nine years in backend..."),
      # ...
  ]
  ```

### 5. Run the script
From the project root:
```bash
cd mvp && uv run python outputs/{slug}/{slug}.py
```
Use a generous timeout (~5 min) — Supertonic is CPU-bound and each turn takes a few seconds. On first run of the project ever, the model weights download (~260 MB).

### 6. Play the audio
```bash
paplay mvp/outputs/{slug}/{slug}.wav
```
Fallback order if `paplay` is missing: `ffplay -nodisp -autoexit ...`, then `aplay ...`, then `xdg-open ...`. Do not retry on the same player if it succeeded silently.

### 7. Report to user
2-3 sentence summary:
- Topic, duration in seconds, voices used.
- Paths: the `.py` and `.wav`.
- One-liner replay command: `paplay mvp/outputs/{slug}/{slug}.wav`.

## Constraints
- Dialogue text MUST be English (Supertonic's English voice quality is best; non-English is out of scope for this skill).
- Slug MUST be lowercase ASCII with hyphens only — no spaces, no diacritics, no underscores.
- `DIALOGUE` MUST be a real Python list literal, not a JSON-encoded string.
- Always `mkdir -p mvp/outputs/{slug}/` before writing the `.py` file.
- One play per invocation. Do NOT auto-regenerate if the user just wants a replay; tell them the replay command.
